1.3. Week 3 Worklog

Week Objectives:

  • Hiểu và thực hành quản lý mã nguồn bằng Git.
  • Triển khai thành công một website mẫu lên máy chủ EC2.
  • Làm quen với khái niệm mạng ảo AWS VPC và Subnet.
  • Sử dụng CloudWatch để giám sát tài nguyên hệ thống.
  • Báo cáo tiến độ và rèn luyện kỹ năng làm việc nhóm.

Tasks to implement this week:

Day Task Start Date End Date References
Mon - Tìm hiểu và thực hành sử dụng Git để quản lý source code (clone, commit, push, pull)
- Học quy trình làm việc nhóm cơ bản (Git Flow)		 04/05/2026 04/05/2026 https://git-scm.com/doc
Tue - Tìm hiểu cơ bản về kiến trúc mạng ảo AWS VPC (Virtual Private Cloud)
- Phân biệt Public Subnet và Private Subnet		 05/05/2026 05/05/2026 https://docs.aws.amazon.com/vpc/
Wed - Kết nối vào EC2 qua SSH để chuẩn bị môi trường
- Cài đặt Web Server (Apache/Nginx) và cấu hình các package cần thiết		 06/05/2026 06/05/2026
Thu - Thực hành Deploy:
+ Kéo source code (pull) từ Git về EC2
+ Thực hành deploy website mẫu lên EC2 và kiểm tra hoạt động qua Public IP		 07/05/2026 07/05/2026
Fri - Tìm hiểu dịch vụ AWS CloudWatch
- Cấu hình theo dõi tài nguyên EC2 (CPU, Network, Disk) và tạo cảnh báo (Alarm)		 08/05/2026 08/05/2026 https://docs.aws.amazon.com/cloudwatch/
Sat - Tổng hợp lại các task đã hoàn thành trong tuần
- Báo cáo tiến độ công việc trực tiếp với mentor và trao đổi rút kinh nghiệm		 09/05/2026 09/05/2026

Week Results:

  • Đã biết cách sử dụng Git để quản lý, phiên bản hóa mã nguồn và sẵn sàng cho việc làm việc nhóm.
  • Nắm được các khái niệm mạng nền tảng trên AWS (VPC, Subnet, Internet Gateway) để thiết lập môi trường bảo mật.
  • Tự tay thiết lập môi trường Web Server và deploy thành công một website mẫu lên máy chủ EC2, website có thể truy cập được từ internet.
  • Biết cách sử dụng CloudWatch để theo dõi sức khỏe và hiệu suất của EC2 instance, thiết lập cảnh báo khi CPU quá tải.
  • Nâng cao kỹ năng giao tiếp, làm việc nhóm và quy trình báo cáo tiến độ dự án chuyên nghiệp với mentor.
